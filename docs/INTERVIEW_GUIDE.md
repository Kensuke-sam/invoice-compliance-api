# Interview Guide: invoice-compliance-api

## 30秒サマリー

- 請求データを受け取り、監査フラグとレビューコメントを返す API。
- ルール判定とAI補助を分離して、説明責任を維持。
- BtoB業務システムを意識した設計で、実務適用のイメージが湧きやすい。

## 3分説明の順番

1. 課題設定: 請求レビューの優先度付けが人依存で遅れる。  
2. 設計: 厳密な入力型 + ルールベース + AIコメントで責務を分けた。  
3. 安全性: AIを最終判定にしないことで、誤判定時のリスクを制御した。  
4. 品質: テスト、型、Lint、CI で変更時の回帰を防いだ。  
5. 拡張余地: 承認フロー、監査ログ、ベンダー別しきい値の追加。

## 深掘り質問に対する答え方

- なぜ AI を補助に限定したか: 監査領域では説明責任が必須だから。  
- なぜ JSON 明細を使ったか: 明細の可変構造に対応しつつ、バリデーション可能にするため。  
- 学生としてアピールする点: 技術だけでなく、運用上の責任分界まで設計に落としたこと。
